Você é uma IA especialista em desenvolvimento full-stack, arquitetura de software, geoprocessamento, IA aplicada e criação de MVPs para hackathon. Quero que você desenvolva um projeto completo chamado **LinhaMap**, uma **Plataforma Preditiva de Trafegabilidade Rural para Ariquemes/RO**.

## 1. Contexto do projeto

O LinhaMap tem como objetivo prever e monitorar o risco de bloqueio em estradas vicinais de Ariquemes e região. A plataforma deve cruzar previsão meteorológica, chuva acumulada, características da via, declividade, histórico de incidentes e relatos da comunidade para indicar, com até 7 dias de antecedência, quais trechos rurais têm maior probabilidade de se tornarem intransitáveis.

O foco é substituir a manutenção reativa por uma manutenção preventiva, ajudando produtores rurais, cooperativas, transportadoras, piscicultores, pecuaristas e a Secretaria de Obras a tomarem decisões baseadas em dados.

## 2. Problema que o sistema resolve

No período chuvoso, linhas rurais de Ariquemes ficam intransitáveis e comprometem o transporte de leite, peixe, gado, café, hortaliças e insumos. Muitas vezes a manutenção só acontece depois que caminhões atolam ou produtores já tiveram prejuízo.

O sistema deve ajudar a reduzir:

* Perdas de produção perecível;
* Aumento do custo logístico;
* Isolamento de propriedades;
* Falta de dados para decisões públicas de manutenção;
* Demora na identificação de trechos críticos.

## 4. Objetivo do MVP

Criar uma plataforma web funcional que classifique trechos rurais em quatro níveis de risco para os próximos 7 dias:

* Baixo;
* Médio;
* Alto;
* Crítico.

A plataforma deve usar dados públicos, dados simulados quando necessário e relatos da comunidade para gerar um **Índice de Trafegabilidade** de 0 a 100 por trecho.

### 6.2 Mapa de risco

Criar uma tela principal com mapa usando Leaflet.

O mapa deve exibir linhas vicinais segmentadas em trechos. Cada trecho deve ter uma cor conforme o nível de risco:

* Verde: baixo;
* Amarelo: médio;
* Laranja: alto;
* Vermelho: crítico.

Ao clicar em um trecho, exibir um painel lateral ou modal com:

* Nome do trecho;
* Linha vicinal;
* Nível de risco;
* Índice de Trafegabilidade de 0 a 100;
* Previsão de chuva para 7 dias;
* Chuva acumulada nas últimas 72h;
* Declividade;
* Quantidade de relatos recentes;
* Explicação do score;
* Recomendações de ação.

### 6.3 Índice de Trafegabilidade

Criar uma função de cálculo do score de 0 a 100.

Sugestão de lógica:

* Quanto maior a chuva prevista, maior o risco;
* Quanto maior a chuva acumulada em 72h, maior o risco;
* Quanto maior a declividade, maior o risco;
* Quanto mais relatos recentes, maior o risco;
* Quanto maior a severidade das denúncias, maior o risco;
* O score final deve ser convertido para nível textual:

  * 0 a 24: baixo;
  * 25 a 49: médio;
  * 50 a 74: alto;
  * 75 a 100: crítico.

A função deve retornar também uma explicação textual, por exemplo:

“Risco alto devido à chuva acumulada de 78 mm nas últimas 72h, previsão de chuva intensa para os próximos dias e 3 relatos recentes de lama e atolamento.”

### 6.4 Cadastro de denúncia colaborativa

Criar uma tela/formulário onde o produtor possa enviar uma ocorrência.

Campos:

* Nome do produtor ou morador;
* Telefone ou WhatsApp;
* Linha vicinal;
* Localização GPS;
* Descrição do problema;
* Upload de foto;
* Categoria da ocorrência;
* Severidade;
* Data e hora;
* Status:

  * Aberta;
  * Em análise;
  * Resolvida.

A localização pode ser capturada automaticamente pelo navegador usando geolocation, mas também deve permitir preenchimento manual.

Após o envio, salvar no banco de dados e atualizar o painel de ocorrências.

### 6.5 Classificação automática da denúncia

Criar uma função no backend que receba a imagem e a descrição da denúncia.

Caso exista chave de API de visão, usar IA multimodal para classificar a imagem.

Caso não exista chave de API, criar fallback com regras simples:

* Se descrição contém “lama”, categoria = lama;
* Se contém “buraco”, categoria = buraco;
* Se contém “ponte”, categoria = ponte danificada;
* Se contém “atolou” ou “atolamento”, categoria = atolamento;
* Se contém “erosão”, categoria = erosão.

Retornar JSON no seguinte formato:

{
"categoria": "lama",
"severidade": "alta",
"confianca": 0.87,
"justificativa": "A descrição indica acúmulo de lama e dificuldade de passagem."
}
